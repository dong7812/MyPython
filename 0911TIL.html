자료구조와 알고리즘

1. 이분 탐색 (Binary Search)

기본 개념

정의: 정렬된 배열에서 특정 값을 빠르게 찾는 알고리즘. 배열을 반으로 나누어 탐색 범위를 줄여가며 찾는다.

시간 복잡도: O(log N)

사용 사례: 정렬된 데이터에서 특정 값을 찾거나(결정 문제), 특정 조건을 만족하는 경계를 찾는 문제(최적화)(예: 이분 탐색으로 최적화 문제 해결).

간단한 코드 예시

def binary_search(arr, target):
    left, right = 0, len(arr) - 1
    while left <= right:
        mid = (left + right) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    return -1  # 찾지 못한 경우

# 사용 예시
arr = [1, 3, 5, 7, 9, 11]
target = 7
print(binary_search(arr, target))  # 출력: 3 (인덱스)

주의 사항

  • 정렬 필요: 이분 탐색은 정렬된 배열에서만 동작. 입력 데이터가 정렬되지 않았다면 먼저 정렬(O(N log N))해야 함.
  • 인덱스 오버플로우: mid = (left + right) // 2 계산 시, left + right가 매우 클 경우 오버플로우 가능. 이를 피하려면 mid = left + (right - left) // 2로 계산.
  • 반복문 vs 재귀: 위 코드는 반복문 방식. 재귀 방식도 가능하지만, 스택 오버플로우를 조심.
  • 경계 조건: left <= right 조건을 정확히 설정하지 않으면 무한 루프 가능.

2. 분할 정복 (Divide and Conquer)

기본 개념

정의: 문제를 작은 하위 문제로 나누고, 각 하위 문제를 해결한 뒤 결과를 합쳐 전체 문제를 해결.

시간 복잡도: 문제에 따라 다름(예: 병합 정렬 O(N log N), 퀵 정렬 평균 O(N log N)).

사용 사례: 병합 정렬, 퀵 정렬, 이진 탐색, 고속 푸리에 변환(FFT) 등.

간단한 코드 예시 (병합 정렬)

def merge_sort(arr):
    if len(arr) <= 1:
        return arr
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])  # 왼쪽 분할
    right = merge_sort(arr[mid:])  # 오른쪽 분할
    return merge(left, right)  # 병합

def merge(left, right):
    result = []
    i = j = 0
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1
    result.extend(left[i:])
    result.extend(right[j:])
    return result

# 사용 예시
arr = [5, 2, 8, 1, 9, 3]
print(merge_sort(arr))  # 출력: [1, 2, 3, 5, 8, 9]

주의 사항

  • 재귀 깊이: 재귀 호출이 깊어지면 스택 오버플로우 발생 가능. Python은 기본 재귀 깊이 제한이 있으므로 큰 입력에서는 주의.
  • 메모리 사용: 병합 정렬처럼 추가 메모리(임시 배열)를 사용하는 경우, 메모리 효율성을 고려.
  • 문제 구조: 모든 문제가 분할 정복에 적합하지 않음. 하위 문제가 독립적이어야 효과적.
  • 합병 과정: 병합 단계에서 로직 오류(예: 인덱스 범위 초과) 주의.

3. 스택 (Stack)

기본 개념

정의: LIFO(Last In, First Out) 구조. 마지막에 추가된 데이터가 먼저 제거됨.

시간 복잡도: 삽입/삭제 O(1), 탐색 O(N).

사용 사례: 함수 호출 스택, 괄호 검사, DFS(깊이 우선 탐색) 등.

간단한 코드 예시

class Stack:
    def __init__(self):
        self.items = []
    
    def push(self, item):
        self.items.append(item)
    
    def pop(self):
        if not self.is_empty():
            return self.items.pop()
        return None
    
    def is_empty(self):
        return len(self.items) == 0

# 사용 예시
stack = Stack()
stack.push(1)
stack.push(2)
print(stack.pop())  # 출력: 2
print(stack.pop())  # 출력: 1

주의 사항

  • 빈 스택 처리: pop() 호출 시 스택이 비어있는지 확인 필수.
  • 메모리 효율: Python 리스트를 사용할 경우 동적 배열로 구현되므로, 메모리 재할당 비용 고려.
  • 스택 오버플로우: 재귀 호출에서 스택을 사용할 경우, 깊은 재귀는 메모리 초과 가능.
  • 스레드 안전성: 멀티스레드 환경에서는 스택 접근 시 동기화 필요.

4. 큐 (Queue)

기본 개념

정의: FIFO(First In, First Out) 구조. 먼저 추가된 데이터가 먼저 제거됨.

시간 복잡도: 삽입/삭제 O(1), 탐색 O(N).

사용 사례: BFS(너비 우선 탐색), 작업 스케줄링, 대기열 관리.

간단한 코드 예시

from collections import deque

class Queue:
    def __init__(self):
        self.items = deque()
    
    def enqueue(self, item):
        self.items.append(item)
    
    def dequeue(self):
        if not self.is_empty():
            return self.items.popleft()
        return None
    
    def is_empty(self):
        return len(self.items) == 0

# 사용 예시
queue = Queue()
queue.enqueue(1)
queue.enqueue(2)
print(queue.dequeue())  # 출력: 1
print(queue.dequeue())  # 출력: 2

주의 사항

  • 효율적인 구현: Python에서는 collections.deque를 사용. 리스트로 pop(0)를 사용하면 O(N)이라 비효율적.
  • 빈 큐 처리: dequeue() 시 큐가 비어있는지 확인.
  • 메모리 관리: 큰 데이터셋에서는 큐 크기 제한을 고려.
  • 원형 큐: 고정 크기 큐가 필요한 경우, 원형 큐(Circular Queue) 구현 고려.

5. 우선순위 큐 (Priority Queue)

기본 개념

정의: 각 요소에 우선순위가 부여되며, 높은 우선순위의 요소가 먼저 처리됨. 힙(Heap)으로 구현됨.

시간 복잡도: 삽입/삭제 O(log N), 최소/최대값 조회 O(1).

사용 사례: 다익스트라 알고리즘, 허프만 코딩, 작업 스케줄링.

간단한 코드 예시

import heapq

class PriorityQueue:
    def __init__(self):
        self.items = []
    
    def push(self, item, priority):
        heapq.heappush(self.items, (priority, item))  # 우선순위 기준으로 삽입
    
    def pop(self):
        if not self.is_empty():
            return heapq.heappop(self.items)[1]  # 아이템 반환
        return None
    
    def is_empty(self):
        return len(self.items) == 0

# 사용 예시
pq = PriorityQueue()
pq.push("task1", 2)
pq.push("task2", 1)
print(pq.pop())  # 출력: task2 (우선순위 1이 더 높음)

주의 사항

  • 최소 힙: Python의 heapq는 최소 힙. 최대 힙은 우선순위에 음수를 붙여 구현.
  • 우선순위 정의: 동일 우선순위 처리 방식(예: FIFO 순서 보장 여부) 확인.
  • 복잡한 객체: 객체를 저장할 경우, 비교 연산(__lt__)을 정의하거나 튜플로 우선순위 명시.
  • 빈 큐 처리: pop() 시 비어있는 경우 예외 처리.

6. 링크드 리스트 (Linked List)

기본 개념

정의: 노드(데이터 + 다음 노드 포인터)로 구성된 선형 자료구조. 동적 크기 조절 가능.

시간 복잡도: 삽입/삭제 O(1) (노드 위치 알 때), 탐색 O(N).

사용 사례: 스택/큐 구현, 동적 메모리 관리, 그래프의 인접 리스트.

간단한 코드 예시

class Node:
    def __init__(self, data):
        self.data = data
        self.next = None

class LinkedList:
    def __init__(self):
        self.head = None
    
    def append(self, data):
        new_node = Node(data)
        if not self.head:
            self.head = new_node
            return
        current = self.head
        while current.next:
            current = current.next
        current.next = new_node
    
    def display(self):
        current = self.head
        while current:
            print(current.data, end=" -> ")
            current = current.next
        print("None")

# 사용 예시
ll = LinkedList()
ll.append(1)
ll.append(2)
ll.append(3)
ll.display()  # 출력: 1 -> 2 -> 3 -> None

주의 사항

  • 메모리 관리: 노드 생성/삭제 시 메모리 누수 주의.
  • 포인터 오류: next 포인터를 잘못 설정하면 리스트가 끊어지거나 무한 루프 발생.
  • 탐색 비효율: 인덱스로 직접 접근 불가. 탐색은 O(N).
  • 더블 링크드 리스트: 양방향 이동이 필요한 경우 고려.

7. 해시 테이블 (Hash Table)

기본 개념

정의: 키-값 쌍을 저장하는 자료구조. 해시 함수를 사용해 키를 인덱스로 변환.

시간 복잡도: 평균 삽입/삭제/탐색 O(1), 최악 O(N) (충돌 심할 때).

사용 사례: 빠른 조회(딕셔너리), 캐싱, 빈도수 계산.

간단한 코드 예시

# Python의 dict는 해시 테이블 기반
hash_table = {}

# 삽입
hash_table["apple"] = 5
hash_table["banana"] = 3

# 조회
print(hash_table.get("apple", 0))  # 출력: 5
print(hash_table.get("orange", 0))  # 출력: 0 (기본값)

# 삭제
del hash_table["banana"]

# 키 존재 확인
print("apple" in hash_table)  # 출력: True

주의 사항

  • 충돌 처리: Python의 dict는 내부적으로 충돌을 처리하지만, 직접 구현 시 개방 주소법(Open Addressing)이나 체이닝(Chaining) 고려.
  • 해시 함수: 키를 고르게 분포시키는 좋은 해시 함수 필요.
  • 메모리 사용: 해시 테이블은 메모리를 많이 사용할 수 있음.
  • 불변 키: Python dict는 키로 불변 객체(문자열, 튜플 등)만 사용 가능.

8. 공통 학습 팁과 주의 사항

Python의 내장 모듈 활용

  • collections.deque: 큐, 스택 구현에 효율적.
  • heapq: 우선순위 큐 구현.
  • dict: 해시 테이블로 빠른 조회 가능.

기타 주의 사항

  • 시간/공간 복잡도 분석: 각 자료구조/알고리즘의 복잡도를 이해하고, 문제에 맞게 선택.
  • 디버깅: 인덱스 오류, 포인터 오류, 빈 자료구조 처리 등에 주의.
  • 문제 풀이 연습: LeetCode, HackerRank 등에서 관련 문제(예: "Two Sum" for 해시 테이블, "Merge Sort" for 분할 정복)를 풀어보며 익히기.
